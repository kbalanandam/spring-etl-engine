# Epic E - Portability and packaged-run guidance

## Summary

Epic E covers product usability outside a single developer workstation. It keeps defaults portable, clarifies packaged execution, makes preserved job bundles easier to run consistently across environments, and tracks low-risk documentation/governance improvements such as centralized product-facing naming refresh.

## Scope

This epic is the home for work that:

- improves cross-platform path and default behavior
- documents packaged jar execution clearly for selected jobs
- reduces environment-specific assumptions in examples and defaults
- centralizes rebrandable product-facing naming without changing the repository's technical identity

This epic is **not** the place for runtime scheduling, secret management, or connector expansion by itself.
