# Epic G — Secret injection and secure configuration

## Summary

Epic G covers how the product should consume secrets and sensitive configuration safely in enterprise environments without turning local examples into production guidance.

## Scope

This epic is the home for work that:

- supports secure secret injection from environment variables or approved config sources
- reduces pressure to store credentials directly in committed YAML
- aligns future runtime setup with enterprise deployment expectations

This epic is **not** the place for transport-specific partner security rules or control-plane UI features by itself.
