Runtime assembly Flow composition Compiled flat execution Outputs & observability
Application start Spring Boot prepares the runtime assembly before any scenario, MainFlow, or explicit step plan is bound.
Selected scenario & config set ConfigLoader resolves one selected job-config.yaml and binds the run to one explicit scenario boundary.
Fail-fast validation Checks the chosen source, target, processor, and model contracts before any descriptor projection or step execution begins.
ScenarioRuntimeDescriptor / MAIN_FLOW_PLAN Projects the selected scenario into one MainFlow context with visible subflows, recovery policy, and handoff aliases.
SubFlow composition / SUBFLOW_PLAN Synthesizes ordered subflows, dependency rules, and step membership from the selected scenario descriptor.
Flat step plan / STEP_PLAN Compiles the hierarchy into an explicit ordered Spring Batch step plan and publishes startup-time assembly policy as STEP_PLAN / STEP_READY.
Compiled flat step runtime One explicit step now executes through reader, processor, and writer factories while the runtime preserves mainFlow, subFlow, dependency, handoff, and step-summary context as evidence.
Accepted target / handoff artifact Accepted records are written to the selected target or preserved as the explicit intermediate artifact for the next step.
Rejects / SUBFLOW_SUMMARY Reject artifacts and subflow outcomes are both preserved so downstream phases can be summarized or logged as BLOCKED.
Scenario log / RUN_EVENT / STEP_EVENT / RUN_SUMMARY Scenario-scoped lifecycle logs preserve composition context, recovery policy, counts, artifact paths, and rolled-up flow evidence after MDC is bound to the selected run.
